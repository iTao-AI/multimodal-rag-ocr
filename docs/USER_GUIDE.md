## 🚀 快速开始

### 1. 启动服务

```bash
# 进入后端目录
cd backend

# 启动所有服务
./start_all_services.sh

# 验证服务状态
./status_services.sh
```

### 2. 启动前端

```bash
# 进入前端目录
cd frontend

# 启动开发服务器
npm run dev
```

### 3. 访问系统

打开浏览器：**http://localhost:5173**

## 📚 知识库管理

### 创建知识库

1. 点击左侧导航栏 **知识库**
2. 点击右上角 **新建知识库** 按钮
3. 输入知识库名称和描述
4. 点击 **创建**

**示例**:
```
名称：产品文档库
描述：公司产品手册、使用指南、FAQ 等
维度：1024 (默认)
```

### 查看知识库

1. 在知识库列表点击目标知识库
2. 查看基本信息：
   - 文档数量
   - 向量数量
   - 创建时间
   - 存储大小

### 删除知识库

⚠️ **警告**: 删除后数据无法恢复！

1. 进入知识库详情
2. 点击 **设置** 标签
3. 点击 **删除知识库**
4. 确认删除

## 📄 文档上传

### 上传单个 PDF

1. 进入目标知识库
2. 点击 **上传文档** 按钮
3. 选择 PDF 文件或拖拽到上传区域
4. 等待解析完成

**支持格式**:
- ✅ PDF (推荐)
- ✅ Markdown (.md)
- ⬜ Word (.docx) - 计划中

### 上传进度

上传后可以在 **上传记录** 中查看进度：

| 状态 | 说明 |
|-----|------|
| ⏳ 等待中 | 等待处理 |
| 🔄 解析中 | 正在提取文本 |
| 🔢 嵌入中 | 正在生成向量 |
| ✅ 完成 | 处理成功 |
| ❌ 失败 | 处理失败 |

### 文档管理

**查看文档列表**:
- 文档名称
- 上传时间
- 页数
- 状态

**文档操作**:
- 📖 预览内容
- 🗑️ 删除文档
- 📊 查看统计

### 批量上传

1. 点击 **批量上传**
2. 选择多个 PDF 文件 (最多 10 个)
3. 等待批量处理

**注意**: 批量上传总大小不超过 500MB

## 💬 智能对话

### 开始对话

1. 点击左侧导航栏 **对话**
2. 选择目标知识库
3. 在输入框输入问题
4. 按 Enter 或点击发送

### 对话功能

**流式输出**:
- 答案逐字显示
- 首字延迟<500ms
- 可随时停止

**来源展示**:
- 显示引用文档
- 显示相似度分数
- 点击可查看详情

**历史对话**:
- 自动保存
- 支持切换会话
- 支持清空历史

### 对话示例

**示例 1: 简单问答**
```
用户：如何使用这个系统？

AI: 使用这个系统需要以下步骤：
1. 安装依赖：pip install -r requirements.txt
2. 配置环境：编辑 .env 文件
3. 启动服务：./start_all_services.sh
4. 访问前端：http://localhost:5173

来源：README.md (相似度：0.92)
```

**示例 2: 多轮对话**
```
用户：支持哪些文件格式？

AI: 目前支持 PDF 和 Markdown 格式...

用户：PDF 有大小限制吗？

AI: 单个 PDF 最大 50MB...
```

### 高级功能

**调整参数**:
- Top K: 召回文档数量 (默认 10)
- 阈值：相似度阈值 (默认 0.5)
- 重排序：是否启用 (默认关闭)

**模型选择**:
- Qwen3-VL-Plus (推荐)
- GPT-4o (可选)

## 🔍 检索测试

### 测试检索效果

1. 点击左侧导航栏 **检索测试**
2. 选择知识库
3. 输入测试 query
4. 点击 **测试**

### 查看结果

**检索结果**:
- 文档列表
- 相似度分数
- 内容预览

**统计信息**:
- 检索时间
- 召回数量
- 平均分数

### 参数调优

**Top K**:
- 小 (5): 精确，可能遗漏
- 中 (10-20): 平衡 (推荐)
- 大 (50+): 全面，噪声多

**阈值**:
- 高 (0.7+): 严格，相关性强
- 中 (0.5-0.7): 平衡 (推荐)
- 低 (0.3-0.5): 宽松，召回多

**测试技巧**:
1. 从简单 query 开始
2. 逐步增加难度
3. 对比不同参数效果
4. 记录最佳配置

## 📋 最佳实践

### 文档准备

**推荐**:
- ✅ 清晰的文档结构
- ✅ 适当的章节划分
- ✅ 合理的文档大小 (<20MB)
- ✅ 文本型 PDF (非扫描版)

**避免**:
- ❌ 过大的文档 (>50MB)
- ❌ 扫描版 PDF (OCR 效果差)
- ❌ 混乱的结构
- ❌ 过多的图片

### 知识库组织

**按主题分类**:
```
产品文档库/
├── 用户手册
├── API 文档
├── FAQ
└── 最佳实践
```

**按部门分类**:
```
公司知识库/
├── 技术部
├── 产品部
├── 运营部
└── 人事部
```

### 问答技巧

**好问题**:
- ✅ 具体明确："如何重置密码？"
- ✅ 包含上下文："在登录页面如何重置密码？"
- ✅ 使用专业术语："API 认证流程是什么？"

**避免**:
- ❌ 过于宽泛："告诉我所有信息"
- ❌ 缺少上下文："这个怎么用？"
- ❌ 模糊表述："那个功能"

### 性能优化

**检索优化**:
1. 合理设置 Top K (10-20)
2. 调整阈值 (0.5-0.7)
3. 启用重排序 (精度优先)
4. 使用元数据过滤

**响应优化**:
1. 启用流式输出
2. 设置合理的 max_tokens
3. 调整温度参数
4. 使用缓存

## ❓ 常见问题

### Q1: 上传失败怎么办？

**可能原因**:
- 文件过大
- 网络问题
- 服务异常

**解决方案**:
1. 检查文件大小 (<50MB)
2. 检查网络连接
3. 查看服务状态：`./status_services.sh`
4. 查看日志：`tail -f logs/pdf_extraction.log`

---

### Q2: 检索结果不相关？

**可能原因**:
- 知识库内容不足
- 阈值设置过高
- Query 表述不清

**解决方案**:
1. 补充相关文档
2. 降低阈值 (0.5 → 0.3)
3. 增加 Top K (10 → 20)
4. 优化 Query 表述

---

### Q3: 回答速度慢？

**可能原因**:
- LLM 响应慢
- 检索慢
- 网络延迟

**解决方案**:
1. 启用流式输出
2. 检查 Milvus 状态
3. 启用缓存
4. 降低 max_tokens

---

### Q4: 答案不准确？

**可能原因**:
- 检索结果不相关
- Prompt 设计问题
- 模型理解偏差

**解决方案**:
1. 检查检索结果
2. 调整重排序
3. 优化 Prompt 模板
4. 更换模型

---

### Q5: 如何备份数据？

**Milvus 数据备份**:
```bash
# 导出数据
python backup_milvus.py export --output backup.json

# 导入数据
python backup_milvus.py import --input backup.json
```

**配置文件备份**:
```bash
# 备份 .env
cp backend/.env backend/.env.backup

# 备份前端配置
cp frontend/.env frontend/.env.backup
```
